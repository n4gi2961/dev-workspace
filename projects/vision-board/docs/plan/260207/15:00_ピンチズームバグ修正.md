ピンチズーム focal-point バグ修正 + ボードサイズ拡大                                                                                                  
   
     Context                                                                                                                                               
                                                                                                                                                         
     ピンチズームが「2本指の中間点」ではなく「ボード中心」を起点としてしまうバグが残っている。また、ボードサイズ（1920x1080）が小さすぎて、ズームアウト
     時にボード外に移動→タップ判定がないエリアに入ってスタックする問題がある。

     ---
     バグ原因分析: ピンチズームが focal-point ベースにならない理由

     根本原因: Transform Origin の誤解

     React Nativeの transform は要素の中心 (transform-origin: center) を基準に適用される。

     canvas Animated.View のスタイル:
     left: screenWidth / 2 - CANVAS_WIDTH / 2,  // 要素の左端
     top: screenHeight / 2 - CANVAS_HEIGHT / 2,  // 要素の上端
     transform: [{ translateX }, { translateY }, { scale }]
     // ↑ scale は要素の中心を起点に適用される

     キャンバスローカル座標 (px, py) のスクリーン位置は:
     screenX = screenWidth/2 + translateX + (px - CANVAS_WIDTH/2) * scale
     screenY = screenHeight/2 + translateY + (py - CANVAS_HEIGHT/2) * scale

     現在のバグコード (BoardCanvas.tsx:187-196):
     const canvasOriginX = screenWidth / 2 - CANVAS_WIDTH / 2;  // ← 要素の左端！
     const pointX = (focalX - canvasOriginX - savedTx) / savedScale;
     //             = (focalX - screenWidth/2 + W/2 - savedTx) / savedScale
     //                                         ^^^^
     //                         この +W/2 が余計。W=1920 → 960pxのオフセット

     正しい計算:
     // Transform origin は要素の中心 = screenWidth/2, screenHeight/2
     const pointX = (focalX - screenWidth / 2 - savedTx) / savedScale;
     const pointY = (focalY - screenHeight / 2 - savedTy) / savedScale;
     translateX = focalX - screenWidth / 2 - pointX * newScale;
     translateY = focalY - screenHeight / 2 - pointY * newScale;

     canvasOriginX が「要素の左端」なのに、transform origin は「要素の中心」。この差分 CANVAS_WIDTH / 2 = 960px が丸ごとオフセットとなり、focal-point
     計算が完全に破綻している。

     ---
     実装ステップ

     Step 1: ボードサイズ 5倍拡大

     変更箇所:
     - BoardCanvas.tsx:43-44 — CANVAS_WIDTH = 9600, CANVAS_HEIGHT = 5400
     - BoardCanvas.tsx:88-92 — 初期スケール 0.5 → 0.1 (同等の初期表示を維持)
     - NodeToolbar.tsx:12-13 — 同じ定数を更新

     Step 2: ピンチズーム focal-point 計算修正

     変更箇所: BoardCanvas.tsx:185-196

     // 修正前（バグ）
     const canvasOriginX = screenWidth / 2 - CANVAS_WIDTH / 2;
     const canvasOriginY = screenHeight / 2 - CANVAS_HEIGHT / 2;
     const pointX = (focalX - canvasOriginX - savedTranslateX.value) / savedScale.value;

     // 修正後
     // Transform origin = 要素の中心 = (screenWidth/2, screenHeight/2)
     const pointX = (focalX - screenWidth / 2 - savedTranslateX.value) / savedScale.value;
     const pointY = (focalY - screenHeight / 2 - savedTranslateY.value) / savedScale.value;
     translateX.value = focalX - screenWidth / 2 - pointX * newScale;
     translateY.value = focalY - screenHeight / 2 - pointY * newScale;
