# Vision Board — プロダクト企画書

> 最終更新: 2026-02-06

---

## 1. プロダクト概要

### アプリ名
- **Polalist** (Polaris + List) — 人生の北極星（指針）となるリスト

### (アプリ名予備候補)
- **Asterize** (Asterism + Realize) — 星々（日々の行動）を繋いで、夢を具現化する

### 一言コンセプト
**「夢を可視化し、小さな行動を積み上げて実現するアプリ」**

### ミッション
従来の「貼って終わり」のビジョンボードを、毎日触れて・行動して・変化を感じるインタラクティブな体験に進化させる。ユーザーが自分の夢を日常的に意識し、具体的な行動に落とし込み、その積み重ねを実感できるプラットフォームを提供する。

---

## 2. 解決する課題

### 従来のビジョンボードの問題点

| 問題 | 詳細 |
|------|------|
| **作って終わり** | 壁やアプリに画像を並べた後、見返さなくなる |
| **行動に繋がらない** | 「眺める」だけで具体的なアクションに落とし込めない |
| **努力が見えない** | 日々の取り組みが目に見える形で蓄積されない |
| **始めるハードルが高い** | 白紙のボードから何を貼ればいいかわからない |
| **モバイル体験の欠如** | デスクトップ向けのグリッド表示がスマホ画面では認知負荷が高い |

---

## 3. 提供価値（バリュープロポジション）

### 3つの柱

#### 柱1: 夢を毎日「見る」仕組み
スマホの小画面を逆手に取り、ビジョンボードの画像を**1枚ずつスライドショー形式**で表示。1つの目標に没入する「セリアル・フォーカス」で、夢との心理的距離を毎日縮める。

#### 柱2: 行動で「変わる」フィードバック
ルーティンを完了するたびに画像が**モノクロからカラーへ変化**。「自分の行動で世界が色づく」という即時的な達成感を提供し、習慣形成を心理学的にサポートする。

#### 柱3: 積み重ねが「残る」可視化
完了したタスクが**星として物理的に積み上がる**（Star Stack）。長期的な努力を目に見える資産に変え、「やめたらもったいない」という保有効果で継続を後押しする。

## 4. ターゲットユーザー

### ペルソナA: 自己変革層（Goal-Driven）
- **属性**: 20〜35歳、目標設定に関心が高い
- **動機**: 本気で自分を変えたい、夢を叶えたい
- **課題**: 立てた目標を忘れる、モチベーション維持が難しい
- **響くワード**: 「叶える」「変わる」「積み上げる」「最初の一歩」
- **彩度変化の意味**: 努力の証明
- **Star Stackの意味**: 成長の記録

### ペルソナB: 美的享受層（Aesthetic-Driven）
- **属性**: 18〜30歳、ビジュアル感度が高い
- **動機**: 美しいものに囲まれた日常を過ごしたい
- **課題**: 毎日の生活にワクワクがない、丁寧な暮らしへの憧れ
- **響くワード**: 「眺める」「彩る」「心地よい」「お気に入り」
- **彩度変化の意味**: 日課を楽しむリズム
- **Star Stackの意味**: コレクション的な楽しさ

### マーケティングとの連動
Instagramの発信トーンとテンプレートのカテゴリでペルソナを選択。ディープリンク経由のテンプレートに紐づくメタデータ（`category: goal | aesthetic`）でオンボーディングのトーンを切り替え、各層に最適な初期体験を提供する。

## 5. コア機能

### 5-1. ビジョンボード（キャンバス）
| 項目 | 内容 |
|------|------|
| **目的** | 目標や理想の未来を画像・テキストで自由にレイアウト |
| **仕組み** | ピンチズーム、パン操作、ドラッグ&リサイズでノードを自由配置 |
| **ノード種類** | 画像ノード（URL、座標、サイズ、角丸）、テキストノード（内容、フォント、色） |
| **画像形状** | FREE / ORIGINAL / SQUARE / LANDSCAPE / PORTRAIT |
| **編集機能** | テキスト編集ポップアップ（色・フォント・サイズ）、レイヤー順序変更、削除 |

### 5-2. フォーカスモード（セリアル・フォーカス）
| 項目 | 内容 |
|------|------|
| **目的** | スマホ画面で1つの目標に没入させる |
| **仕組み** | ボード上の画像を1枚ずつ縦スワイプで表示 + ルーティンチェック |
| **心理学的根拠** | 直列的情報提示による認知負荷軽減。親指の可動域を活用した垂直スワイプ |
| **UI要素** | VisionBoardCard表示、ルーティンチェックパネル、縦進捗インジケーター |
| **特殊効果** | タップ時のリップルエフェクト（完了フィードバック） |

### 5-3. 彩度変化システム
| 項目 | 内容 |
|------|------|
| **目的** | ルーティン完了の即時報酬を提供 |
| **仕組み** | 未完了=モノクロ画像、ルーティン完了で段階的にカラーへ変化 |
| **段階** | 初期: 彩度0%（完全モノクロ）→ アクション完了ごとに彩度上昇 |
| **心理学的根拠** | ツァイガルニク効果（未完成→完成への欲求）、目標勾配効果（完成に近づくほど加速） |
| **減衰ペナルティ** | 3日間放置で彩度低下開始（損失回避によるモチベーション維持）。猶予期間と復帰ボーナスあり |

### 5-4. Star Stack（星の積み上げ）
| 項目 | 内容 |
|------|------|
| **目的** | 長期的な努力を目に見える資産として可視化 |
| **仕組み** | タスク完了1回 = 星オブジェクト1個が瓶に物理演算で落下・蓄積 |
| **物理演算** | 弾み、衝突、不安定なバランス。 |
| **心理学的根拠** | 自己効力感の増大、保有効果によるロックイン、イケア効果（自分の労力の結晶） |
| **モチーフ** | 星砂を小さな瓶に詰めるイメージ。Atomic Habitのクリップボックス戦略と同概念 |

### 5-5. ルーティン管理
| 項目 | 内容 |
|------|------|
| **目的** | 目標達成のための日々の行動を管理 |
| **仕組み** | ボードに紐づくルーティンの作成・チェック・トラッキング |
| **データ** | タイトル、色、曜日設定（activeDays）、履歴（日付ごとのbooleanマップ） |
| **連携** | チェック → 彩度変化トリガー + Star Stack蓄積 |

### 5-6. テンプレートシステム
| 項目 | 内容 |
|------|------|
| **目的** | 「白紙の恐怖」を解消し、1タップでボードを始められる |
| **仕組み** | 完成されたビジョンボードをテンプレートとして配布。ディープリンクで直接適用 |
| **カテゴリ** | 丁寧な暮らし、フィットネス、起業家、Y2K、ミニマリズムなどトレンド反映 |
| **成長ループ** | パワーユーザーがUGCとしてテンプレート共有 → 新規ユーザー獲得 |
| **イケア効果** | テンプレートから始め、自分の写真に置き換える過程で所有感が発生 |

## 6. ユーザージャーニー

```
[初回] テンプレートを選ぶ → すぐにボードが完成
  ↓
[毎朝] フォーカスモードで目標画像を1枚ずつ見る
  ↓
[日中] ルーティンをチェック → 画像がカラーに → 星が積まれる
  ↓
[夜]  Star Stackで「今日の積み上げ」を確認
  ↓
[数週間後] 自分の写真に置き換え始める → 完全に自分のボードに
  ↓
[数ヶ月後] 瓶いっぱいの星 = 自分の努力の結晶 → やめられない
```

## 7. ゲーミフィケーション設計

### 二層構造

| 時間軸 | メカニズム | 心理的効果 | UI表現 |
|--------|-----------|-----------|--------|
| **短期（毎日）** | 彩度変化 | 即時報酬（ドーパミン系） | モノクロ写真 → カラー　＋　低い確率で流星のエフェクトと強い触覚フィードバックで報酬系を刺激する |
| **長期（毎月）** | 物理積み上げ | 資産形成感（セロトニン/オキシトシン） | 星が瓶に溜まる |

### なぜ二層が必要か
- **短期報酬だけ**: 新鮮味が薄れると効果が減衰
- **長期報酬だけ**: 日々の行動への即時フィードバックがない
- **二層の相乗効果**: 短期で「今日やる理由」を、長期で「続ける理由」を提供

### 減衰と復帰のバランス
```
活動中:  彩度100% ──────────────────
                                    ＼
3日放置:  段階的に低下               ＼──── 彩度50%
                                          │
復帰時:  復帰ボーナスで50%回復 ────────── 彩度75%
         → 残りは自力で完全カラーへ
```

## 8. 成長戦略（Product-Led Growth）

### テンプレート駆動ファネル

```
[認知] SNS（Instagram/TikTok）
  │  インフルエンサーが完成されたビジョンボードを投稿
  │  彩度変化・Star Stackのアニメーションが映える
  ↓
[獲得] ディープリンク
  │  投稿内リンクからテンプレート適用済みでアプリ起動
  │  「白紙の恐怖」ゼロ
  ↓
[アクティベーション] 初回ルーティン完了
  │  最初のチェックで彩度変化を体験
  │  「やれば変わる」を実感
  ↓
[定着] 個人化
  │  自分の写真に入れ替え → イケア効果
  │  Star Stackが積み上がる → 保有効果
  ↓
[拡散] UGCテンプレート共有
     パワーユーザーが自分のボードをテンプレート化して発信
     → 新規ユーザーの [認知] へ循環
```

## 10. 技術スタック概要

### モバイル版

| 項目 | 選定 | 選定理由 |
|------|------|----------|
| フレームワーク | Expo SDK 52+ | New Architecture標準、OTA更新、EAS Build |
| ルーティング | expo-router v4 | ファイルベース、ディープリンク自動化 |
| スタイル | NativeWind v4 | Tailwind記法流用、学習コスト最小 |
| アニメーション | Reanimated 3 | JSI経由60fps、彩度変化・Star Stack表現 |
| ジェスチャー | react-native-gesture-handler | ドラッグ&ドロップ、スワイプ操作 |
| リスト表示 | FlashList | FlatListの5-10倍高速 |
| 画像 | expo-image | BlurHash、ディスクキャッシュ |
| ストレージ | AsyncStorage + MMKV | 通常データ + 機密データ（暗号化） |
| バックエンド | Supabase | 認証、PostgreSQL、リアルタイム、RLS |

### アーキテクチャ

```
vision-board/（pnpm monorepo + Turborepo）
├── apps/
│   ├── web/          # Next.js Web版
│   └── mobile/       # Expo React Native版
└── packages/
    ├── shared/       # 型定義・定数・ユーティリティ（100%共有）
    └── supabase/     # Supabase抽象化（90%共有）
```

### データモデル

```
Users (Supabase Auth)
  └── Profiles (id, email, dark_mode)
       └── Boards (id, name, user_id)
            ├── Nodes (id, type[image|text], x, y, width, height, data)
            └── Routines (id, title, color, activeDays, history)


*備考:画像用ストレージとしてCloudflare R2を導入予定*
```

## 付録: 競合との差別化

| | 従来のビジョンボードアプリ | Vision Board |
|---|---|---|
| **操作** | 画像を並べて終わり | 毎日開いてルーティンと連動 |
| **表現** | 静的な壁紙 | モノクロ→カラーの動的変化 |
| **達成感** | なし | 彩度変化 + Star Stack |
| **始めやすさ** | 白紙から作る | テンプレートで1タップ開始 |
| **用途** | 見る専用 | 見る + 行動する + 感じる |
| **継続性** | 3日で飽きる | 二層ゲーミフィケーションで定着 |
